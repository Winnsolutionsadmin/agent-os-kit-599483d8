The interim bracket motif

Until a name is chosen, the mark is the bracket. It is a deliberate design motif, not a blank to fill: the company is being named, and that is on-brand. The mark never appears naked; on first appearance in any artifact it locks up with the one canonical descriptor line.

[PROJ-X]

[ PROJ-X ], the trusted companion for the AI transition.

[PROJ-X] all caps · reads as a token
[Name] title case · softer

square brackets only · never parentheses · never angle brackets · one placeholder everywhere, never a rotating set · owner picks one casing and holds it

Do

  • Use one placeholder, the same bracket, across every artifact.
  • Pair with the descriptor line on first appearance in each document.
  • Keep the treatment quiet and typographic, at the weight a real wordmark would occupy.
  • In the room, say "the product" or read the descriptor, never "bracket name bracket".

Do not

  • Fill the brackets with a favorite name in any shared artifact.
  • Put "agent OS", the prior engineering framework, or any retired working title inside the brackets.
  • Commission an ornate logo for a name that does not exist yet.
  • Let the placeholder read as indecision: the name is the one call we will not rush.
retirement plan The moment a name is chosen, the bracket motif retires in a single coordinated pass across every artifact, and the domain, handles, and intent-to-use filing lock that same week. One artifact may keep the story: the owner memo can preserve the "we were deliberately unnamed" beat as brand lore, since being-named-is-part-of-the-story is itself on-brand.
