## Status conventions

- **complete** = backlog row done, commit sha in the description (audit trail).
- **in progress** = artifact drafted, sitting at a human gate (founder review / counsel); the gate is named in the description.
- **to do** = open or blocked; blocked rows say "BLOCKED on <x>" in the description. ClickUp has no blocked status on this list; the backlog's Blocked-by column is authoritative.
- Human gates are never closed by agents: FOUNDER/counsel-gated rows stay in progress or to do until Jarred or counsel acts. proposeDone + proof, never silent close.

## How loops update the board

1. Loop picks the top unblocked row from its W*/tasks.md.
2. Executes, commits the ARTIFACT to the workstream folder.
3. Updates the tasks.md row (status + artifact path) and commits: backlog first, always.
4. Syncs the matching ClickUp subtask: status change + description update carrying artifact path and commit sha (PUT /task/{id}).
5. New rows added to tasks.md get new subtasks under the same anchor, same "W<n>.<m> <title>" naming.
6. Readiness scores never sync to ClickUp; they are internal prioritization only.

API access pattern (token on the mini; never printed): `ssh mini 'python3 -' < script.py`, token via regex `pk_[0-9A-Za-z_]+` over `~/.openclaw/secrets/*` + `~/.openclaw/.env`, endpoints `api.clickup.com/api/v2/list/901819360023/task` (GET with `?subtasks=true&include_closed=true` for dedup, POST with `parent` for subtasks, PUT `/task/{id}` for updates).
